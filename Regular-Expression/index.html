Regular Expression

Posted on 2017-11-22 | In Universal

We use Regular Expression to match strings,for example to judge whether the string is a legal email address or telephone number.I have summarized common usage of Regular Expression, as follows:

1 Basic Qualifiers:

“\d“：Matches a numeric character. Equivalent to [0-9].
“\w“: Matches a letter, number, or underscore. Equivalent to [A-Za-z0-9_].
“\s“: Matches a blank character, including spaces, tabs, pages breaks, and so on. Equivalent to [\f\n\r\t\v].
“\b“: Matches a word boundary, such as “is” in the matching string “It is RegExp”,can be written as \bis\b, here \b identifies whether the word “is” is the word boundary.

2 Special Qualifiers:

“^“: Matches the start of the input string. If you set the Multiline property of a RegExp object, ^ also can match the position after \n or \r.
“$“: Matches the end of the input string. If you set the Multiline property of a RegExp object, $ also can match the position before \n or \r.
“.“: Matches any single character other than \n. To match any character including \n, we can use a pattern like (.|\n).
“?“: Matches the previous subexpression zero or once. For example, th(is)? Matches “th” or “this”. ? is equivalent to {0,1}.

3 Range Qualifiers:

“{n}“: Matches n times. For example, o{2} does not match “o” in “god”, but can match double “o” in “good”.
“{n,}“: Matches at least n times. For example, o{2,} does not match “o” in “god”, but matches all “o” in “goooood”. Thus,we can find that o{1,} is equivalent to o+. o{0,} is equivalent to o*.
“{n,m}“: Matches at least n times and at most m times. For example, o{1,3} will match the first three “o” of “gooooood”. o{0,1} is equivalent to o?.

4 Greedy Qualifiers:

“*“: Matches the previous subexpression zero or more times.Greedy match,matches all in priority. For example, go* matches “g” and “goo”. *is equivalent to {0,}.
“+“: Matches the previous subexpression one or more times.Greedy match, matches all in priority. For example, go+ matches “go” and “goo”, but can not match “g”. + is equivalent to {1,}.

5 Non-Greedy Qualifiers:

when ? follows immediately after any of the other qualifiers(*, +,?, {n}, {n,}, {n, m}), the match patterns are non-greedy.
“*?“: Matches any times, but match as little as possible.
“+?“: Matches one or more times, but match as little as possible.
“??“: Matches zero or once, but match as little as possible.
“{n}?“: Matches n times, but match as little as possible.
“{n,}?“: Matches at least n times, but match as little as possible.
“{n,m}?“: Matches at least n times and at most m times, but match as little as possible.

6 Advanced Qualifiers:

“(express)“: Matches express, and capture the text to the automatically named group.
“(?express)“: Matches express and capture the text into a group named name.
“(?:express)“: Matches express, does not capture the matched text.
“(?=express)“: Matches the position before express. For example, “I am learning RegExp”, using.+(?=RegExp),the result is “I am learning”.
“(?<=express)“: Matches the position after express. For example, “I am learning RegExp”,using(?<=am>).+,the result is “learning RegExp”.
“(?!express)“: Matches the position unequal to express before. For example, “123abc”,using\d{3}(?!\d) to match.
“(?<!express)“: Matches the position unequal to express after. For example, “abc123”,using(?<![0-9])123 to match.

7 Antonyms Qualifiers:

Transfor the lowercase letter to uppercase,and the meaning is opposite.
“\D“: Matches any character that is not number.
“\W“: Matches any character that is not letter, number, or underscore.
“\S“: Matches any character that is not a blank character.
“\B“: Matches position that is not the beginning or end of a word.

8 Other Qualifiers:

“x|y“: Matches x or y.
“[xyz]“: Matches any one of the characters contained. For example, [abc] can match “a” in “apple”.
“[^xyz]“: Matches any character that is not included. For example, [^abc] can match “p” in “apple”.
“[a-z]“: Matches any character in the specified range. For example, [a-z] can match any lowercase alphabetic character from “a” to “z”.
“[^a-z]“: Matches any character that is not within the specified range. For example, [^a-z] can match any character that is not in the “a” through “z” range.
